# 02. Process Protocol

본 문서는 KSTATION Manufacturing 프로젝트의 **운영 파이프라인**를 정의합니다.
최종 수정: 2026-04-10

---

## 0. 프로젝트 구조

| 프로젝트              | 경로                          | 현재 단계                |
| ----------------- | --------------------------- | -------------------- |
| **케이필<가제>** (OEM 제조) | `01_New_Product/`           | OEM 답변 대기 → 패키징 → 론칭 |
| **공통**            | `00_Overview/`, `용어사전.md` | 프로젝트 연동              |

---

## 1. 작업 단계 분기


**[Research] 리서치 단계**
- 커뮤니티 관찰, 페인포인트 수집, 경쟁사 분석

**[Design] 제품 설계 / OEM 협의 단계**
- OEM 협의, 스펙 확인
- 기존 LINE FILL과 동일한 스펙으로 이해

**[Marketing] 마케팅 / 콘텐츠 단계**
- 세그먼트별 콘텐츠 제작, 채널 운영, 트랙 반응 모니터링


---

## 2. OEM 협의 파이프라인

**Phase 1 — 스펙 수치 파악 (제품 페이지 공개용)**
- G', pH, Buffer 타입, COA 발행 여부 → OEM 답변 요청
- 공유된 수치만 `01_New_Product/02_Product_Design/01_스펙시트.md`에 [Confirmed] 기재

**Phase 2 — 인증/품질 문서 확인**
* COA 항목, Lot 편차 → `01_New_Product/04_Operations/02_COA_체크리스트.md` 적용

---

## 3. 콘텐츠 트랙 운영 파이프라인

**배포 단계**
- Track A ~ C × 뉴비/고관여층 초기 콘텐츠 동시 배포
- 채널: TikTok/Instagram (뉴비) + Reddit (고관여층)

**신호 수집 단계 (론칭 후 1-6주)**
- 저장/공유/댓글/DM/클릭/구매 데이터 → `01_New_Product/05_Logs/02_커뮤니티_신호_로그.md` 기록

**집중 전환 단계 (6주 후)**
- 반응 높은 트랙 콘텐츠 증산, 반응 낮은 트랙 조정
- 결정 → `01_New_Product/05_Logs/01_의사결정_로그.md` 기록

- `01_커뮤니티_페인포인트.md`: 신규 신호 관찰 즉시 추가 (날짜 포함)
